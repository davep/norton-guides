Troubleshooting


Question : "When I link with RTLink I get a lot of "double defined"
           warnings !  What am I doing wrong ?"

Answer   : "You're doing nothing wrong. Those warnings are absolutely
           normal.  They occur because I rewrote a part of the General
           Terminal, so it also works in graphical mode. You always should be
           getting 24 of such warnings. The ALERT is also rewritten."


Question : "When I link I get "undefined functions", __gtSLR and
           __gtModalRead, __gtIsColor and a few other. What's wrong ?"

Answer   : "You forgot to include the /NOE switch. If you don't use this
           switch, some functions which are used from the standard GT API
           can't be found (aren't searched for). Always include this switch."


Question : "How do I lose that logo of Ferns' Graphical Library ? I
           hate the waiting for three seconds!"

Answer   : "Simple, just register and you will get the registered version,
           which (of course) doesn't have the logo, or the waiting."


Question : "Everytime I use SAVE SCREEN in a graphical mode, it takes ages
           before the screen is saved..what's wrong ?"

Answer   : "You are doing nothing wrong, but in graphical mode a lot of
           information on the screen has to be saved. For instance a save
           screen in 1024x768x256 mode needs to use almost 800kB of VMM !!!
           (In textmode this is only 35kB). Always watch out with SAVE
           SCREENS !"
