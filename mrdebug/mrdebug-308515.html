 1.00.75e - Bug Fix - Release
 Bug Fix Release
──────────────────────────────────────────────────────────────────────────────

 This addressed the following problems:

 1. Work undertaken for more closer compatibility with Provision:Windows and
    the replacements that PVW make.

 2. In the dbfList screen if you pressed <ALT><G> within the view window and
    then pressed <Esc> the border would not return to the correct window.

 3. Changed behaviour of the STEP functions, found under the Options->Display
    sub-menu and added an extra function.

 4. Change of behaviour for the F4 and Alt-0 keys. They will now use the STEP
    functions, should they not be empty.

 5. A few problems with animate when in full screen mode

 6. Switching to and from different screen modes.

 7. Fixed GPF that would result from mode switching if Tab was pressed.


 And changed the following:

 Under the Options->Display Menu:

    Step Entry   removed and replaced with Before Step
    Step Exit    removed and replaced with After Step

 and added:

    Step Restore

    <F4> now switches modes if app is in different mode.


 These should aid in the debugging of graphical applications.

 For further information refer to the entries in the relevant places.
