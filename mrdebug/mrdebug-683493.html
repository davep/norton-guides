 Protected mode compatible
 MrDebug works well in protected mode
──────────────────────────────────────────────────────────────────────────────

 MrDebug has been tested with Blinker 3 running in protected mode. And our
 comments are that Blinker 3 must have been designed with MrDebug in mind!

 This has to be the fastest method of running MrDebug. With no memory
 penalties and no waiting for overlays to swap in and out with those huge
 30,000 line programs.

 If you are using Blinker 3, then MrDebug takes advantages of this fact. If
 you include Blinker's SWPRUNCMD, then MrDebug will use this function for DOS
 Access - giving you as much as 610K in a DOS Shell.

 MrDebug is also fully compatible with Causeway and Exospace - so no matter
 which protected mode linker you already have, MrDebug will work with your
 protected mode linker.
