 If I Shell to DOS and change the video mode, the MrDebug screen is garbled.
──────────────────────────────────────────────────────────────────────────────

 MrDebug doesn't know that you have changed the video mode and so displays
 what it think is correct.

 You can get around this problem by telling MrDebug what video mode you want
 to keep MrDebug in. Under the Options menu is a sub-menu called Display
 Setup, this sub-menu contain an item called Screen Mode Change. This item
 allows you to enter a video mode number (in decimal) that MrDebug will use
 to keep the video mode.

 If you save your settings, this mode number will be used to start up MrDebug
 each time you restart your application.
