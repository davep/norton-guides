 About MrD
 A restart option for MrDebug
──────────────────────────────────────────────────────────────────────────────

 MrD.EXE allows you to restart an application that has been linked with
 MrDebug.

 When an application runs and has MrDebug linked in, normally there is no way
 of restarting the application. It is simply a case of exiting MrDebug and
 running the program again.

 With MrD.exe, MrDebug will recognize the fact the MrD.exe is being used and
 offer you a Restart option on the File menu.

 Note: This option is NOT recommended for real mode programs. MrD.exe will
       take a massive 15 K away from conventional memory. If you get problems
       in real mode because you are using MrD.exe don't phone technical
       support - the problem is down to memory - for debugging purposes move
       to protected mode.

 Note: If you have about 15K of contiguous UMB space available then it is
       possible to load high Mrd.exe. By preceding MrD <Appname> with an LH,
       This will load the Restart module into an UMB thus not wasting 15K of
       conventional memory.
