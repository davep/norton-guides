 Colour Syntax Highlighting
 What goes on
──────────────────────────────────────────────────────────────────────────────

 By default, the colour syntax highlighting of the source code is toggled to
 on. You may change this either because you do not like colour syntax
 highlighting or because you find that it slows down MrDebug too much.

 The toggles to turn this on or off can be found under the Source appearance
 sub-menu of the Options menu. You also have control of the colours used by
 MrDebug for the source code colour syntax highlighting.

 We also give you the ability to define which words appear as keywords.

 MrDebug looks for it's keywords in a file called MRDEBUG.WRD. By default,
 there is no such file, but we give you three examples files that you may
 edit to suit your purposes and copy one of them to MRDEBUG.WRD. The examples
 files are:

        COMMANDS.WRD - contains most CA-Clipper commands
        KEYWORD.WRD  - contains most CA-Clipper commands, functions,
                       statements etc
        CONTROL.WRD  - contains only CA-Clipper statements.
