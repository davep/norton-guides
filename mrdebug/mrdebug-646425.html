 About MrDump
 What is MrDump
──────────────────────────────────────────────────────────────────────────────

 MrDump is a crash file creator that lists various items that may be of
 interest to you after a program has crashed. Quite often, with a standard
 error system, the information reported is not enough to track down a bug or
 a problem.

 MrDump will help you here by dumping, to a file, the error and error object
 details, the contents of the stack, information about workareas, a dump of
 the screen and a list of the variables - in scope or all.

 Ideally, a call to MrDump should be made inside your errorsystem so that
 when an error occurs and the errosystem is called, MrDump is also called and
 a dump file will be created.

 Please Note: MrDump.LIB must be linked in as a FILE. So don't forget to
              include the .LIB extention. ie. FI MrDump.LIB
