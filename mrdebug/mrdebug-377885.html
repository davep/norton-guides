 Create Dump File
 Create a log file.
──────────────────────────────────────────────────────────────────────────────

 This option will produce a listing of current memory variables at each level
 of the program at the time of choosing the option.

 When you choose this option a dialog box will appear asking for the name of
 a file for the output to be placed. The default file name is the name of the
 .EXE being run, with an extension of .MDL

    █▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀ Log Filename Definition ▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀▀█
    █ Filename: F:\SOURCE\MRDEBUG\NEWDBF\TEST.MDL                          █ 
    █▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄▄█ 
                                                                             

 If you wish to use another name, then simply type the name of the file and
 an extension and press ◄──┛.

 If you do not specify a drive or directory, then the file will be created in
 the current directory.

 Information contained within the log file include a listing of all variables
 in each of the functions calls contained within the call stack; a listing of
 the tables open with information about indexes, relations and filters; and a
 dump of the application screen.

 An example is shown below:
