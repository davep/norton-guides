 I'm having memory problemsin Real Mode. Is there anything I can do?
──────────────────────────────────────────────────────────────────────────────

 Memory is short and MrDebug is having problems running eh?. Well the first
 thing to do is try a different link file.

 MrdMax.lnk is useful in circumstances where memory is low. MrdMax.lnk will
 overlay as much of MrDebug as possible, thus reducing the root size of your
 application and hopefully, allowing you to debug successfully.

 If MrdMax.Lnk does still not resolve your problems, we suggest that you only
 compile the .PRG(s) with the /b switch that are relevant - i.e. those
 .PRG(s) that directly effect the problem that you are trying to track down.
