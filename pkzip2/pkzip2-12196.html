  -k    Retain original .ZIP file date
  -k-   Set .ZIP file date to the current date (override
          configuration file)

  Normally, when PKZIP updates an existing .ZIP file by either adding to it
  or deleting from it, the .ZIP file date is changed to the current date.
  If the -k option is specified on the command line the .ZIP file date will
  not be altered.

  This option may be made the default, see PKZIP Configuration.

  If PKZIP is configured to keep the original date in the configuration
  file, this may be overridden temporarily by use of the -k- option.
