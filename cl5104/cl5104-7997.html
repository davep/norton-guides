  BOF()             Set to .T. on LastRec() + 1 access.

  Clipper Version  1.29 dtd 04/15/91.

  Problem Statement

  BOF() will return .T. under the following example:
       USE test
       GO LASTREC() + 1
       ? BOF()   // => .T.

  Cause

  Unknown.

  Work Around(s)

  Don't 'Jump' to LastRec() + 1 and test for BOF(). Skip works ok.
