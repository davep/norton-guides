   På EGA of VGA skærme er der ofte problemer med de danske
   specialkarakterer, specielt ¥ og ¢.   Dette skyldes at EGA og VGA
   adaptorer i modsætning til Monocrome adaptorer gemmer sit tegnsæt i RAM.
   Det er så brugeren der bestemmer hvordan dette tegnsæt skal se ud.   Fra
   og med DOS 3.3 er der defineret en række såkaldte Codepage's, og
   problemet er så blot reduceret til at vælge den rigtige.   For danske
   brugere vil det typisk være to forskellige der kan have interesse.   CP
   (Code Page) 850 er en såkaldt Multi Language code page.   Den indeholder
   specialtegn fra så og sige alle sprog, men da disse tegn er lavet på
   bekostning af "rammetegnene" er CP 865 efter min overbevisning mere
   interresant.   CP 865 er en nordisk code page, og indeholder derfor både
   specielle Norske, Svenske og Danske tegn.
   For at vælge CP 865 indsættes efterf¢lgene linier i AUTOEXEC.BAT og
   CONFIG.SYS:

   CONFIG.SYS

                COUNTRY=045,865,C:\DOS\COUNTRY.SYS
                DEVICE=C:\DOS\DISPLAY.SYS CON = (EGA,865)

   Den f¢rste linie fortæller systemet noget om hvordan datoformatet i de
   forskellige lande er, mens linie 2 fortæller hvilke code page's den
   anvendte skærmadaptor kan anvende.

   AUTOEXEC.BAT

                PATH=C:\;C;\DOS
                MODE CON CODEPAGE PREPARE = ((865) C:\DOS\EGA.CPI)
                MODE CON CODEPAGE SELECT = 865
                C:\DOS\KEYB DK 865 C:\DOS\KEYBOARD.SYS

   Den f¢rste linie er kun  medtaget for at undgå at skulle angive s¢gesti
   til hver enkelt fil.
   Linie 2 forbereder CP 865 til brug, mens linie 3 vælger den.
   Den sidste linie laver opsætning til Dansk tastatur.
   Linie 2 og 3 kan også ses forkortet, hvilket giver samme resultat:

                MODE CON CP PREP = ((865) C:\DOS\EGA.CPI)
                MODE CON CP SEL = 865

   I disse to listninger er kun medtaget kommandoer der direkte vedr¢rer
   problemet med de danske karakterer.   Se andetsteds for komplet
   konfigurering.
